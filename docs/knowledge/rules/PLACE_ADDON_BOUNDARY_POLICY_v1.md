## 1. Purpose

This policy defines the hard boundary between what CORE may include as soft environmental resonance and what requires the future PLACE add-on. It also defines what is forbidden at all phases.

---

## 2. PLACE is a future add-on — not active CORE runtime

```text
PLACE is a planned Darrow Code add-on product.
It is not active in CORE production runtime.
It is not active in any current route, prompt, schema, or PDF template.

PLACE becomes active only after:
1. MAP0 — provider capability audit for astrocartography/relocation
2. MAP-PLACE — mapping provider data to PLACE rule docs
3. Explicit product and runtime approval
4. Implementation phase separate from KB2-PLACE
```

## 3. CORE boundary — what CORE may include

CORE may include a small environmental resonance note if chart patterns strongly support it.

### 3.1 Allowed in CORE

```text
✅ Broad environmental archetype — the kind of environment a pattern tends to resonate with
✅ 1–3 illustrative place examples — named cities/regions as symbolic reference, clearly marked non-prescriptive
✅ Non-ranked soft mention — no "best city," no "top 5," no personalized ranking
✅ Environmental framing — coastal vs structured vs expansive vs quiet, as archetype only
✅ Non-personalized examples — general illustrations, not selected for this specific user's chart + goals
```

### 3.2 Blocked in CORE

```text
❌ Maps of any kind
❌ Astrocartography lines
❌ Relocated chart calculations or interpretations
❌ City ranking ("your top 3 cities are...")
❌ "Best city" or "worst city" verdicts
❌ Personalized city/country recommendations based on target location inputs
❌ Relocation instructions or advice
❌ Goal-based place matching (that belongs to PLACE add-on)
❌ Practical reality check section (that belongs to PLACE add-on)
```

---

## 4. PLACE add-on boundary — what PLACE may include

### 4.1 Allowed in PLACE add-on (future, after MAP0/MAP-PLACE approval)

```text
✅ User-selected goals (career, love, recovery, creativity, etc.)
✅ User-provided target locations (city/region inputs)
✅ Astrocartography / planetary line analysis
✅ Relocated chart analysis for specified targets
✅ City/region comparison based on chart resonance + user goals
✅ Final place strategy — framed as symbolic resonance + practical reality check
✅ Mandatory practical reality check section
✅ Goal-based place archetypes (from PLACE_USE_CASE_ARCHETYPES_v1.md)
✅ Ranking within user-selected shortlist (not global "best city" claim)
```

### 4.2 Blocked in PLACE add-on at all phases

```text
❌ Relocation advice: "move here," "leave your current city," "buy property there"
❌ "Best city in the world" or global ranking claims
❌ Love / money / fame / health / safety guarantees from location
❌ Medical, legal, financial, or immigration guidance
❌ Compatibility claims from comparing two people's place charts
❌ Astrocartography without exact birth time
❌ Relocated chart without exact birth time and verified coordinates
❌ Any claim framing the place as fated, karmic, or destined
❌ Claims that a location is dangerous or harmful to the person
```

## 5. Practical reality check — mandatory in PLACE add-on

Every PLACE add-on output that discusses place options must include or link to a practical reality check.

```text
MANDATORY SECTION: Practical Reality Check

"Symbolic place resonance is one input among many. Real-world feasibility
must always be weighed alongside it."

Required reality-check factors:
  - visa / residency rights and complexity
  - language and communication access
  - employment / business market conditions
  - family ties and social network implications
  - cost of living and financial practicality
  - personal safety
  - healthcare access and quality
  - legal and financial situation
  - personal preference, lifestyle tolerance, and values
  - climate and physical environment tolerance
  - cultural fit and community access

The symbolic resonance of a place informs the question. Real-world factors answer it.
```

This section is not optional in any PLACE output. It may be brief; it may not be absent.

---

## 6. The "no move/leave/buy" rule

```text
Darrow Code must NEVER:
  - tell a user to move to a city
  - tell a user to leave their current location
  - tell a user to buy property in a specific place
  - tell a user to immigrate
  - advise a user to change relationship, career, or legal status based on place resonance
  - frame any of the above as what the chart "says they must do"
```

This rule applies at all phases — CORE, PLACE add-on, and any future cross-linked add-on.

---

## 7. Illustrative example rule

CORE and PLACE may use illustrative city/region examples. Every illustrative example must be:

```text
✅ Clearly marked as an example, archetype, or illustration
✅ Non-ranked and non-personalized
✅ Framed as symbolic reference, not recommendation
✅ Accompanied by wording like: "places like...", "as an archetype...", "for illustration...",
   "this kind of environment...", "think of cities such as..."
```

Illustrative examples must NOT be:

```text
❌ Presented as personalized recommendations ("your city is...")
❌ Ranked or scored ("this is the best option for you")
❌ Presented as certain ("you will do well in X")
❌ Selected from the user's actual target location list without MAP-PLACE processing
```

## 9. Claim policy — all PLACE content

### Allowed claim register

```text
symbolic place-resonance observation
environmental archetype framing
non-ranked illustrative example
field emphasis description (supportive / amplifying / pressuring / softening)
conditional language ("may," "tends to," "can," "is likely to emphasize")
```

### Forbidden claim register

```text
love / money / fame / health / safety guarantees
"best / worst" verdicts
relocation instructions
move / leave / buy / immigrate advice
legal / medical / financial / safety advice
fate / destiny / karma certainty
"this is your city"
rankings without user goal + target input
city-specific claims without verified coordinates and calculation
astrocartography without exact birth time
```

# Add-on Sample Synthesis (reference only)

**Reference only.** These old samples are style/structure DNA — they do **not** override
`module-content-contracts.md`, `material-assembly-readiness.md`, the do-not-claim registry, or
current launch scope. Where a sample violates a contract, the pattern is **rewritten safe** below.

## Global add-on canon (from DARROW CODE ADD-ON SYSTEM CONCEPT)

- Add-ons are **focused applied orientation layers** — one life domain, practically usable.
- **Practical usability over abstraction.** Clarity without pressure.
- Human warmth, validation, reassurance — **desire and ambition are allowed**.
- **Support without taking authority** over the client's life (no commands).
- **Before/after = a shift in experience**, never a guaranteed outcome.
- **Dinner Table Test:** every line must sound natural to an intelligent adult — not inflated
  jargon. Lead with a lived moment; data confirms quietly after (recognition-first).
- Not therapeutic, not predictive, not dependency-based. No new product/subscription scope now.

## Shared structure DNA (all samples)

`orientation → archetype/philosophy → 3–5 domain sections (each: pattern → scenario →
mechanism → protocol) → closing integration`. Strong single **archetype/metaphor** per module;
recurring "Protocol" callouts; a warm **closing/integration** page.

---

## LOVE — from DYAD CODE (solo) + Architect Edition

- **Opening:** the relationship *contract* / inner architecture; "how you let people in."
- **Structure:** relational archetype → love style → emotional need → conflict loop → intimacy protocol → closing.
- **Metaphors (safe):** Fortress / Sanctuary, Safe Harbor, Secret Garden, "relationship firewall", shared silence, decompression.
- **Phrasing:** boundaries-as-care, privacy, non-verbal connection, attachment without blame.
- **Protocols:** decompression ritual, repair sequence after conflict, intimacy pacing.
- **Closing:** "ageing together" → reframed as *the relationship you're building toward*.
- **Inspires prompt/schema:** archetype + love_style + emotional_needs + conflict_loop + intimacy_protocol.
- **Inspires renderer:** archetype eyebrow, protocol callouts, warm closing page.
- **Discard/gate:** ❌ synastry, partner comparison, "ideal partner", leave/stay commands,
  partner-specific data, Tandem two-person structure → **future compatibility product only**.

## MONEY — from Vector Code

- **Opening:** money as **structure, positioning, leverage, authority** — not income.
- **Structure:** authority archetype → income engine → value & leverage → money friction → negotiation/authority protocol → closing.
- **Metaphors (safe):** System Architect, "build in silence", silent power, hydraulic press (steady force).
- **Phrasing:** value creation, positioning, authority role, hidden strength, workflow.
- **Protocols:** negotiation framing, authority-building behavior, friction-removal routine.
- **Closing:** long-game value orientation.
- **Inspires prompt/schema:** authority_archetype + income_engine + value_and_leverage + money_friction + negotiation_protocol.
- **Discard/gate:** ❌ investment advice, asset allocation %, buy/sell/invest, income/wealth
  guarantees, "this will make money", specific assets. "Old money" only as *symbolic value orientation*.

## BODY — from VITAL CODE

- **Opening:** **capacity vs motivation**; the body as an energy system to calibrate.
- **Structure:** capacity archetype → energy battery → stress/pressure valve → recovery & sleep → practical protocol → closing.
- **Metaphors (safe):** Battery, engine, "decompression chamber" (sleep), pressure valve.
- **Phrasing:** load, pacing, overload signals, recovery rhythm, sensory reset, sleep hygiene.
- **Protocols:** morning/evening routine, SOS/crisis recovery, decompression.
- **Closing:** longevity-as-rhythm (not medical longevity claims).
- **Discard/gate:** ❌ diagnosis, treatment, symptoms/disease, clinical claims (vessels,
  inflammation, autophagy, psychosomatics). "Consult a professional" only as a general disclaimer.
  Carries the verbatim BODY disclaimer.

## YEAR — from Solar Architect (+ Continuum tone)

- **Opening:** the year as **terrain / strategic arc** — "a weather map for your system."
- **Structure:** annual theme → timing phases (quarters) → green & pressure zones → domain timing (work/relationship/energy) → strategic summary → closing.
- **Metaphors (safe):** terrain/arc, weather map, green zones / pressure zones, preparation windows.
- **Phrasing:** timing as orientation not control; signal over noise; decisions stay with the client.
- **Protocols:** when to push vs prepare; window-based decision framing.
- **Discard/gate:** ❌ guaranteed predictions, fixed future dates not in the data, "this will
  happen", fear-based "black swan". Timing only from available transit/solar/BaZi-flow + Personal Year.
  Continuum = **future monthly product**, not current YEAR.

## STYLE — from AESTHETICS CODE

- **Opening:** **visual signature & sensory architecture**; "signal before display, presence before impression."
- **Structure:** aesthetic archetype → material & texture → silhouette & presence → presentation/interface → closing.
- **Metaphors (safe):** Interface Strategy, Stealth Wealth, Visual Silence, tactile authority, "the uniform strategy."
- **Phrasing:** style as interface/boundary/presence; texture, silhouette, grooming, workspace atmosphere.
- **Protocols:** wardrobe-as-uniform system, environment/object curation.
- **Discard/gate:** ⚠️ the sample's **color-palette / stones** section is **gated** — keep the
  *structure* (visual tone direction) but **no named magical colors, no stones as luck/healing/
  protection**, no specific brand prescriptions, until a verified dictionary is approved. Lead with
  texture/material/silhouette/presentation (contract-allowed), not color correspondences.

## PLACE — from Place Code

- **Opening:** **location architecture & spatial alignment**; "context before strategy."
- **Structure:** environment archetype → what regulates → what drains → space protocol (home/workspace/recharge) → closing.
- **Metaphors (safe):** Exoskeleton, Fortress/shell, Captain's Bridge (workspace), Bunker (recharge).
- **Phrasing:** light, silence, materials, thresholds, privacy, sensory architecture, cohabitation.
- **Protocols:** silence protocol, workspace setup, recharge-zone design.
- **Discard/gate:** ❌ astrocartography, exact city/region zones ("Europe/Asia/Miami are your
  zones"), relocation promises, "live here / don't". Reframe to **qualities of space**, relocation
  as *self-reflection*, never a named place.

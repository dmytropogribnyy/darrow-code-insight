# Security policy

## Reporting a vulnerability

Please report suspected vulnerabilities through GitHub's private vulnerability reporting feature when it is enabled for this repository. If that option is unavailable, contact the repository owner through their GitHub profile before sharing technical details. Do not include exploit details, credentials, personal data, or sensitive operational information in a public issue.

Include the affected area, a concise reproduction, the potential impact, and any suggested mitigation. Reports will be reviewed and prioritized according to severity and reproducibility.

## Scope

Useful reports include issues affecting authentication, authorization, protected report access, payment-event handling, data exposure, secret handling, input validation, or document delivery.

Public product documentation does not contain the application's implementation or operational configuration. This repository is not the appropriate place to submit customer information, report contents, payment data, or production diagnostics.

## Supported versions

Security fixes are applied to the current maintained product version. Historical documentation snapshots are not supported.
